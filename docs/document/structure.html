The structure of <mod>.jar mod file:
  1. mod/
    1. Module.class
  2. META-INF/
    1. MANIFEST.MF
  3. resources/
    1. ...
  4. mod.json

mod.Module class

package mod; // Fixed package entry

            public class Module {
                private Module() {}
                public static class Items {
                    public static class /*class name*/{
                        // Fields must be static
                        public static String name; // Item name (required)
                        public static String itemtype; // Item type (required)
                        public static String type; // Item level (default 0)
                        public static int durability; // Item durability (default 1)
                        public static boolean noLevel; // If the item has no level (default true)
                        public static int level; // Item level level number (requied when noLevel == false)
                        public static boolean attack; // canAttack (default true)
                        public static boolean spriteSheet; // If the sprite is using sheet (required)
                        public static int[] sprite; // Sprite position (default {0, 0})
                    }
                }
                public static class Recipes {
                    public static class /*class name*/{
                        // Fields must be static
                        public static String creation; // Creation name (required)
                        public static String[] require; // Array of requiement names (required)
                        public static String type; // Recipe type (default workbench)
                    }
                }
            }

Sprite

In resources, if spriteSheet == false, then it finds with resources/item/<item_name>.png.

META-INF/MANIFEST.MF (MANIFEST)

Not required for now.

resources/

This directory contains the sprite sheets, for standard sprite sheet, at resources/items.png, otherwise, at resources/item/<item_name>.png.

mod.json

The content of this json will be displayed at the "Mods" Menu, only name (mod name) and description (mod description) are required.
